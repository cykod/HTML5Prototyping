What is prototyping?

Wikipedia Says:


A prototype is an early sample or model built to test a concept or process or to act as a thing to be replicated or learned from.

A model to be replicated
or learned from.

Prototype ≠ Final Product
(Usually)

Why a prototype (not a final product)?

  1. #1: There's something you need (or want) to do.

  2. #2: There's something you don't know:

    1. Does anyone care?

    2. Design, User Flow, Interaction

    3. Software architecture - Tools, DBs, Languages

  3. #3: The quicker you learn what you don't know, the quicker you can get your job done.

How to attack a prototype?

#1: Build Only what's needed

#2: Dispose with Ceremony

Pick the Best Tools* for the job and get it done as quickly as possible.

(No authentication, data model, styling as necessary)

#3: Iterate, Iterate, Iterate

Get *something* up ASAP

#4: Be ready to throw it all away

One prototype might not be able to test everything

#5: Decide what's production code

Make sure you apply the normal rigor to those parts

Web Servers

Serve

  • $ gem install serve
  • Go to a directory, run 
    
$ serve      # or:
$ serve 4001 # different port
  • Use all your normal stuff: Sass, Compass, HAML (Additional gems Req'd)
  • For more of a file structure: 
    serve create project   # create a new project in the project directory
  • Can be served live or by running the export command.
  • More details at: http://get-serve.com/

Deploying to Heroku

  • 
$ mkdir serve-test
$ cd serve-test
$ git init
  • Create a Gemfile: 
    
# Gemfile
source 'https://rubygems.org'

gem "serve"
gem "haml"
gem "sass"
  • Create a Procfile for heroku: 
    
web: bundle exec serve $PORT
  • Create any content: 
    
# index.html.haml
!!!5
%html
  %head
    %title Serve Example
  %body
    Here be content
  • Commit and deploy to heroku: 
    $ heroku create serve-exampler
$ git add .
$ git commit -m "Initial Example"
$ git push heroku master

Yeoman

  • "Yeoman is a robust and opinionated set of tools, libraries, and a workflow that can help developers quickly build beautiful, compelling web apps." - Yeoman.io
  • Install via npm, or via (w/ additional notifications): 
    
curl -L get.yeoman.io | bash
  • Front End Development on steroids: Easy boilerplate, testing, components, building
  • Create a new projects 
    
$ mkdir yeoman-test
$ yeoman init        # Ask for some defaults
$ yeoman init --help # See available builders
  • Running the Web Server. Supports LiveReload or a polling process. 
    
$ yeoman server
  • Can easily install components (using web package manage bower) 
    
$ yeoman install bootstrap
$ yeoman install backbone
  • When you're ready to deploy, just run build and deploy your dist/ directory. 
    
$ yeoman build

Data: MongoDB is great for prototypes

  • Easy to install on almost any platform
  • Schema-less - send it data, get data back.
  • Well-supported: Run yourself, on Heroku, or via SaaS.
  • More www.mongodb.org

mongodb-rest

  • $ npm install mongodb-rest
  • Create directory with a config.json to allow CORS requests: 
    
{ "db": {
  "port": 27017,
  "host": "localhost"
},
"server": {
  "port": 3000,
  "address": "0.0.0.0"
},
"accessControl": {
  "allowOrigin": "*",
  "allowMethods": "GET,POST,PUT,DELETE,HEAD,OPTIONS"
},
"flavor": "nounderscore",
"debug":      true
}
  • Go to directory, run 
    
$ mongodb-rest
  • Full access to your MongoDB database via open REST. Awesome.
  • Dangerous as hell.

Example on Heroku

http://cors-demo.herokuapp.com/heroku_app7929607/rest
